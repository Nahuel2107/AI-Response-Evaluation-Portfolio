# Core Criteria for AI Response Evaluation

This guide summarizes the main criteria used to evaluate AI-generated responses.

## 1. Accuracy

Accuracy measures whether the response is factually correct.

A response may have accuracy issues if it:

- Provides incorrect facts
- Uses wrong dates, names, numbers, or definitions
- Makes unsupported claims
- Contradicts the prompt
- Presents speculation as fact

Example:

**Prompt:** What is the capital of Argentina?

**Good response:** The capital of Argentina is Buenos Aires.

**Poor response:** The capital of Argentina is Córdoba.

## 2. Instruction Following

Instruction following measures whether the response follows the user's request.

A response may fail instruction following if it:

- Ignores part of the prompt
- Provides too many or too few items
- Uses the wrong format
- Answers a different question
- Fails to respect user constraints

Example:

**Prompt:** List three fruits.

**Good response:** Apple, banana, and orange.

**Poor response:** Apple.

## 3. Clarity

Clarity measures whether the response is easy to understand.

A response may have clarity issues if it:

- Is confusing
- Is poorly organized
- Uses vague wording
- Includes unnecessary complexity
- Has unclear structure

## 4. Helpfulness

Helpfulness measures whether the response actually helps the user achieve their goal.

A response may be unhelpful if it:

- Is too vague
- Gives irrelevant information
- Does not solve the user's problem
- Lacks useful detail
- Does not answer the main request

## 5. Safety

Safety measures whether the response avoids harmful, dangerous, or inappropriate content.

A response may have safety issues if it:

- Encourages illegal activity
- Gives dangerous instructions
- Provides unsafe medical, legal, or financial advice
- Promotes self-harm or violence
- Fails to warn about risk when necessary

## 6. Reasoning

Reasoning measures whether the logic behind the response is valid.

A response may have reasoning issues if it:

- Uses weak logic
- Makes unsupported conclusions
- Contradicts itself
- Skips important steps
- Does not explain why the answer makes sense

## 7. Written Justification

A good evaluation should include a short, clear justification explaining why one response is better than the other.

A strong justification should:

- Mention the most important criteria
- Be specific
- Avoid vague statements
- Explain the main weakness of the losing response
- Be written in clear English
